## 简介
本文件面向“致良知教育”项目的学习中心系统，围绕课程列表页、课程详情页及其子模块进行系统化技术文档说明。内容涵盖：
- 课程列表页的筛选、排序与展示逻辑
- 课程详情页的模块化架构、数据绑定与交互流程
- 课程数据看板（学习趋势、完成率、成就）的设计与实现
- 今日课程任务的打卡机制、学习进度跟踪与状态管理
- 学习路径规划、进度计算算法与完成状态管理
- 用户认证系统集成与数据同步机制

## 项目结构

## 架构总览
系统采用“页面 + 组件”分层架构，页面负责生命周期与路由，组件负责具体业务与数据绑定。统一请求封装负责鉴权与错误处理，API 配置集中管理后端接口。

```mermaid
sequenceDiagram
participant U as "用户"
participant CL as "课程列表页"
participant API as "API配置"
participant REQ as "统一请求封装"
participant CD as "课程详情页"
participant S as "课程安排"
participant T as "今日课程"
participant D as "课程数据"
U->>CL : 打开课程列表
CL->>API : 读取列表接口路径
CL->>REQ : 发起 GET 请求
REQ-->>CL : 返回课程列表
CL->>U : 渲染卡片与状态标签
U->>CL : 点击课程卡片
CL->>API : 校验报名接口
CL->>REQ : 发起校验请求
REQ-->>CL : 返回校验结果
alt 已报名
CL->>CD : 跳转详情页
else 未报名
CL->>U : 跳转报名页
end
CD->>S : 加载课程安排
S->>T : 切入某天任务看板
T->>API : 读取今日/指定天接口
T->>REQ : 发起 GET 请求
REQ-->>T : 返回任务与进度
T->>U : 展示任务与进度
U->>T : 点击任务并完成
T->>API : 打卡/提交作业
T->>REQ : 发起 POST 请求
REQ-->>T : 返回最新完成率
T-->>CD : 通过事件更新进度
CD-->>U : 进度更新
```

### 课程列表页（课程筛选、排序与展示）
- 状态标签与按钮：根据课程状态、起止时间动态生成“未开营/已开营/已结营/看回放/去学习/去报名”
- 时间格式化：对字符串日期进行安全转换与格式化
- 分页加载：支持下拉刷新与上拉加载，控制 loading 与“已经到底啦”状态
- 路由与报名校验：点击卡片前先校验是否已报名，再决定跳转详情或报名页

```mermaid
flowchart TD
Start(["进入课程列表"]) --> Load["加载状态栏高度<br/>动态设置样式"]
Load --> Fetch["请求课程列表<br/>pageNum/pageSize/status=1"]
Fetch --> Resp{"返回码=200?"}
Resp --> |否| Toast["提示错误并停止刷新"]
Resp --> |是| Merge["合并列表/更新总数"]
Merge --> NoMore{"是否已加载完?"}
NoMore --> |是| Flag["标记 noMore=true"]
NoMore --> |否| IncPage["pageNum++"]
Flag --> Render["渲染课程卡片"]
IncPage --> Render
Render --> Click{"点击卡片?"}
Click --> |是| Check["校验报名接口"]
Check --> Has{"已报名?"}
Has --> |是| Detail["跳转详情页"]
Has --> |否| Enroll["跳转报名页"]
Click --> |否| End(["等待交互"])
```

## 故障排查指南
- 登录失效：统一请求封装检测 401 自动清除 token 并跳转登录页

- 今日课程无课：弹出空状态提示，引导用户休息或回顾
- 课程数据加载失败：提供重试按钮，点击后重新请求
